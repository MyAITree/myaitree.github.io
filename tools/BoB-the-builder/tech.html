Architecture Explanation

The Gift Box Builder application follows a class-based architecture with clear separation of concerns:

Design Patterns Used: Constructor Pattern, Module Pattern, Observer Pattern, Singleton Pattern

Key Components:

  • Product Class: Represents individual products with properties and formatting methods
  • GiftBox Class: Manages the collection of selected products and calculates totals
  • GiftBoxApp Class: Main application controller that handles UI, events, and data flow

Data Flow:

  1. Application initializes and creates sample products
  2. UI renders products and sets up event listeners
  3. User interactions trigger methods that update application state
  4. UI updates reflect changes in state

Event Handling:

// Event delegation pattern example
productsGrid.addEventListener('click', (e) => {
  const productCard = e.target.closest('.product-card');
  if (productCard) {
    const productId = parseInt(productCard.dataset.id);
    this.addProductToBox(productId);
  }
});

The application uses event delegation for efficient handling of dynamic elements and maintains a unidirectional data flow from user interactions to state changes to UI updates.
